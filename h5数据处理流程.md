# H5数据处理流程

## 1. 现在代码中保留的唯一数据读取流程

当前项目已经只保留 `ModelNet40` 的 H5 读取流程，不再保留 `txt`、`npy`、`ScanObjectNN`、`ShapeNet`、`MODELNET10_H5` 等其他入口。

核心调用链如下：

1. `start.py`
   在训练入口中调用 `get_dataset(args, args.dataset)`。
2. `utils.py`
   `get_dataset()` 固定只接受 `MODELNET40_H5`，并固定从 `/root/autodl-tmp/datasets/ModelNet40_h5` 读取数据。
3. `pointnet/pointnet_dataloader_h5.py`
   `H5FolderLoader` 扫描 `root/*/train/*.h5` 或 `root/*/test/*.h5`。
4. `H5FolderLoader.__getitem__()`
   实际进入 `_get_item()`，读取单个 H5 文件。
5. `_get_item()`
   从 H5 文件中读取 `data` 数据集，做归一化，再转成 PointNet 需要的 `[3, N]` 张量格式。
6. `torch.utils.data.DataLoader`
   将单样本组织成 batch，供后续 PointNet 训练和测试使用。

## 3. 标准训练目录要求

`H5FolderLoader` 现在只接受下面这种目录结构：

```text
/root/autodl-tmp/datasets/ModelNet40_h5/
  airplane/
    train/
      xxx.h5
    test/
      xxx.h5
  chair/
    train/
      xxx.h5
    test/
      xxx.h5
  ...
```

扫描规则是：

- 训练集：`root/*/train/*.h5`
- 测试集：`root/*/test/*.h5`

类别标签来源不是 H5 文件内部，而是目录名：

- `.../airplane/train/xxx.h5` -> 类名 `airplane`
- `.../chair/test/xxx.h5` -> 类名 `chair`

然后在 `__init__()` 中自动建立：

- `self.classes: dict[str, int]`

例如：

```python
{"airplane": 0, "bathtub": 1, ...}
```

## 4. 单个 H5 文件的处理过程

单文件处理逻辑位于 [pointnet/pointnet_dataloader_h5.py](/d:/Chain_varify_sampling/pointnet/pointnet_dataloader_h5.py:46)。

### 4.1 读取阶段

执行者：

- `H5FolderLoader._get_item()`

代码动作：

```python
with h5py.File(h5_path, "r") as handle:
    point_set = handle[self.dataset_name][:]
```

这里固定读取：

- 键名：`data`

读出后数据类型：

- `numpy.ndarray`

预期形状：

- `[N, 3]`

预期 dtype：

- `float32`

### 4.2 规范化阶段

执行者：

- `pc_normalize()`

输入：

- `numpy.ndarray`
- 形状 `[N, 3]`
- dtype `float32`

处理内容：

1. 计算质心 `centroid = mean(pc, axis=0)`
2. 点云平移到原点附近 `pc = pc - centroid`
3. 按最大半径缩放到单位球内

输出：

- `numpy.ndarray`
- 形状 `[N, 3]`
- dtype `float32`

### 4.3 PointNet 输入格式转换

执行者：

- `_get_item()`

代码动作：

```python
point_set = point_set.T
return torch.from_numpy(point_set), torch.tensor(label, dtype=torch.long)
```

输出：

- 点云：`torch.Tensor`
- 点云形状：`[3, N]`
- 点云 dtype：`torch.float32`
- 标签：`torch.Tensor`
- 标签形状：标量
- 标签 dtype：`torch.int64`

## 5. `utils.py` 中的数据流

`get_dataset()` 现在只做这一件事：

1. 校验 `dataset == "MODELNET40_H5"`
2. 创建训练集 `H5FolderLoader(root=..., split="train")`
3. 创建测试集 `H5FolderLoader(root=..., split="test")`
4. 包装成 `DataLoader`
5. 返回：

```python
(npoints, coord_dim, num_classes, train_dataset, train_loader, test_loader)
```

返回值的实际含义：

- `npoints`
  当前训练流程沿用上层传入值，默认 `1024`
- `coord_dim`
  固定为 `3`
- `num_classes`
  固定为 `40`
- `train_dataset`
  `H5FolderLoader` 实例
- `train_loader`
  `torch.utils.data.DataLoader`
- `test_loader`
  `torch.utils.data.DataLoader`

## 7. 最终保留下来的“唯一正确流程”

现在代码中保留的唯一流程可以概括为：

1. 从 `/root/autodl-tmp/datasets/ModelNet40_h5` 进入
2. 按 `class_name/train/*.h5` 与 `class_name/test/*.h5` 扫描文件
3. 每个 H5 文件只读取 `data`
4. `data` 必须是 `float32` 的 `[N, 3]`
5. 调用 `pc_normalize()` 归一化
6. 转置成 `[3, N]`
7. 转成 `torch.float32` 张量
8. 标签由目录名映射成 `torch.long`
9. 最后由 `DataLoader` 组成 batch
