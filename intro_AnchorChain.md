## 总结一下链路构建的需求以及损失计算方式

假设目前有AB两个点云，规模相同，一个是真实数据，一个是合成数据

* 首先对两个点云分别进行采样，采样方式K-medien，采样大小为K个点（K点先设置为64），这些点命名为anchor点

* 采样以后，计算每个anchor点下的如下值，形成点特征向量，这样一个anchor点不仅代表自己，还代表【所属的一片区域】：

  ```
  原始坐标
  x，y，z
  
  (下面两个统计暂时不用实现，过段时间再说)
  邻域半径分布统计：最近的16个原始点的距离，提取如下统计量
  均值，标准差，最小值，最大值，25%分位，中位数，75%分位
  
  anchor邻域半径分布统计：最近的16个其他的anchor，提取如下统计量
  均值，标准差，最小值，最大值，25%分位，中位数，75%分位
  ```

*  按照A点云的anchor点采样的时候默认得到的顺序，使用 Sinkhorn方法构建一个B的关联矩阵，为A在B上寻找唯一对应的点，其“对应成本”的计算方法为：

  ```
  把anchor和邻域塞进一个pointnet中（可以调用官方库实现），得到一个embedding，这个pointNet是可以学习的
  然后使用embedding作为计算子，计算代价，形成Sinkhorn矩阵
  在进行相似矩阵硬化，实现一一对应。
  硬化后的矩阵变为排序矩阵P，对B的点进行排序，这样A中的B3N的顺序就对应B中的B3N顺序了
  ```

* 将A，B得到并且排序好的anchor集合，使用模型网络的encoder部分，映射成高维向量。也就是得到

  ```
  映射前：B 3 N
  映射后；B C N
  同时存在两个数据，也就是real_origin, real_project,syn_project,syn_project
  ```

* 得到映射前后的顺序后，按照如下方法计算骨架向量：

  ```
  Step 1：几何描述
  geo_feat = [r(1)...r(n-1), θ1...θ(n-2), gr(1)...gr(n-2), gθ1...gθ(n-3)]
  其中这些量的计算方法如下：
  r为从一个点到下一个点的欧式距离长度
  gr是从一个点到下一个点的下一个点的欧氏距离长度（二阶长度）
  θ为第一个点到第二个点的向量，和第二个点到第三个点的向量的夹角
  gθ为第一个点到第二个点的向量，和第一个点到第三个点的向量的夹角
  
  
  Step 2：feature描述
  feat_feat = [r(1)...r(n-1), θ1...θ(n-2), gr(1)...gr(n-2), gθ1...gθ(n-3)]
  计算方式类似上述情况，但是是在映射后的高维空间计算的，目的是让其映射后的结果在更高维度仍然可以维持结构特点
  
  Step 3：拼接，结合差异项，ε作为可学习的参数
  final=concat(geo_feat,feat_feat,∣geo_feat−feat_feat∣,feat_feat/(geo_feat+ε))
  ```

* 最后根据全局final，计算loss方法如下：

  ```
  L_global =
      λ_g * || g_syn - g_real ||_1
    + λ_f * || f_syn - f_real ||_1
    + λ_Δ * || Δ_syn - Δ_real ||_1
    + λ_ρ * || ρ_syn - ρ_real ||_1
  ```
